# TryHackMe – SOC Level 1 Alert Triage

## Project Description

This project documents my completion of the **SOC Level 1 Alert Triage** room on TryHackMe. The lab provided hands-on experience in the Security Operations Center (SOC) alert triage process, where I learned how to investigate security alerts, determine their severity, identify potential false positives, and prioritize incidents based on risk.

Throughout the lab, I analyzed simulated security events, reviewed available evidence, and applied a structured approach to alert investigation. The project demonstrates practical SOC Level 1 analyst skills, including alert analysis, incident prioritization, and documentation, while following a systematic investigation workflow.

## Objectives

* Understand the role and responsibilities of a SOC Level 1 Analyst.
* Learn the alert triage process.
* Analyze security alerts using available evidence.
* Differentiate between true positives and false positives.
* Assess alert severity and business impact.
* Prioritize incidents based on risk.
* Document investigation findings and recommended actions.

## Lab Overview

During this lab, I worked through multiple alert investigation scenarios that simulated real-world SOC operations. Each scenario required reviewing alert details, gathering relevant evidence, analyzing available information, determining whether the alert represented malicious activity, assigning an appropriate severity level, and documenting the investigation outcome.

The exercises reinforced the importance of following a consistent investigation methodology to ensure accurate and efficient incident handling.

## Investigation Workflow

1. Review the alert details.
2. Validate the alert and identify affected assets.
3. Gather supporting evidence.
4. Analyze indicators and available logs.
5. Determine whether the alert is a true positive or false positive.
6. Assess the severity and potential business impact.
7. Recommend appropriate response actions.
8. Document the investigation findings.
